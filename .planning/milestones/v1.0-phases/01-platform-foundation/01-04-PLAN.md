    ```typescript

    ```

    Create helper functions:
    - `createCheckoutSession(orgId: string, priceId: string, userEmail: string)`: Creates a Stripe Checkout Session in "subscription" mode. Sets `client_reference_id` to orgId (for webhook correlation). Sets success_url to `/billing/success?session_id={CHECKOUT_SESSION_ID}`, cancel_url to `/billing`. If the org already has a stripe_customer_id in the subscriptions table, pass it as `customer`; otherwise let Stripe create a new customer. Allow promotion codes.
    - `createPortalSession(stripeCustomerId: string)`: Creates a Stripe Customer Portal session for self-service management. Return URL = `/billing`.

    **Webhook handler (apps/web/app/api/webhooks/stripe/route.ts):**
    Follow Pattern 5 from RESEARCH.md EXACTLY:
    - POST handler
    - Read raw body with `await request.text()` (NEVER request.json() — Pitfall 6)
    - Verify signature with `stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!)`
    - On verification failure: return 400
    - Handle events:
      - `checkout.session.completed`: Read client_reference_id (orgId), subscription ID, customer ID. Update subscriptions table: set stripe_customer_id, stripe_subscription_id, plan_tier='pro', status='active'.
      - `customer.subscription.updated`: Sync status and cancel_at_period_end from Stripe subscription object. If status changed to 'canceled', update plan_tier back to 'free'.
      - `customer.subscription.deleted`: Set plan_tier='free', status='canceled', clear stripe_subscription_id.
      - `invoice.payment_succeeded`: Update current_period_end from invoice period.
      - `invoice.payment_failed`: Set status='past_due'.
    - For DB operations: use Supabase server client with service_role key (webhook has no user session — must bypass RLS).
    - Return 200 for all processed events.

    **Billing schemas (packages/shared/src/schemas/billing.ts):**

    - `subscriptionStatusSchema`: z.enum(['active', 'canceled', 'past_due', 'trialing', 'incomplete'])
