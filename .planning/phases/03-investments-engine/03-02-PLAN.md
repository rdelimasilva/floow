    1. `getAssets(orgId: string)` — SELECT all assets for org, ordered by ticker
    2. `getPortfolioEvents(orgId: string, assetId?: string)` — SELECT events with optional asset filter, ordered by eventDate desc
    3. `getLatestPrices(orgId: string)` — For each asset, get the most recent asset_price entry. Use a subquery or group approach. Returns Map<assetId, priceCents>.
    4. `getPositions(orgId: string)` — The core query: fetch all assets, for each asset fetch its events and latest price, call computePosition(), return enriched Position objects with asset metadata (ticker, name, assetClass) plus computed fields (quantityHeld, avgCostCents, currentPriceCents, currentValueCents, unrealizedPnLCents, unrealizedPnLPercent, realizedPnLCents, totalDividendsCents). Filter out positions with quantityHeld === 0 (fully sold) from the main view but include them if they have realized PnL.
    5. `getIncomeEvents(orgId: string, months?: number)` — SELECT portfolio_events WHERE event_type IN ('dividend', 'interest', 'amortization') for the last N months (default 12), joined with assets to include ticker/name. IMPORTANT: Query from portfolio_events NOT transactions to avoid INV-07 double-counting (see Pitfall 6 from research).

    Create `apps/web/lib/investments/actions.ts`:
    - 'use server' directive
    - Import createDb, assets, portfolioEvents, assetPrices, transactions, accounts from @floow/db
    - Import createAssetSchema, createPortfolioEventSchema, assertEnv from @floow/shared
    - Import eq, sql from drizzle-orm
    - Import getOrgId from '@/lib/finance/queries'
    - const DATABASE_URL = assertEnv('DATABASE_URL')

    Implement:
    1. `createAsset(formData: FormData)` — Validate with createAssetSchema, insert into assets, revalidatePath('/investments')
    2. `createPortfolioEvent(formData: FormData)` — The critical action implementing INV-07:
       - Validate with createPortfolioEventSchema
       - Define CASH_FLOW_EVENT_TYPES mapping (from research Pattern 3): buy={transactionType:'expense', sign:-1}, sell={transactionType:'income', sign:1}, dividend/interest/amortization={transactionType:'income', sign:1}, split=null
       - Use db.transaction() to atomically:
         a. Insert portfolio event
         b. If event type has cash flow mapping AND totalCents is set:
            - Insert corresponding transaction row (type from mapping, amountCents = sign * |totalCents|, description = "{eventType}: {ticker}", date = eventDate)
            - Atomic balance update on the linked account: sql`balance_cents + ${signedAmount}`
            - Update portfolio event's transactionId to link back for audit trail
       - revalidatePath('/investments'), revalidatePath('/dashboard'), revalidatePath('/transactions')
    3. `updateAssetPrice(formData: FormData)` — Accept assetId + priceCents + priceDate, insert into assetPrices, revalidatePath('/investments')
