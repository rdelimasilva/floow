**A. Create `apps/web/components/finance/create-recurring-dialog.tsx`** (client component)

A modal dialog for creating and editing recurring templates. Uses native `<dialog>` element (matching CreateRuleDialog pattern from Phase 6). Props:

```typescript
'use client'

interface AccountOption {
  id: string
  name: string
}

interface CategoryOption {
  id: string
  name: string
  type: string
}

interface CreateRecurringDialogProps {
  open: boolean
  onClose: () => void
  accounts: AccountOption[]
  categories: CategoryOption[]
  // Optional — for edit mode
  editTemplate?: {
    id: string
    accountId: string
    categoryId: string | null
    type: 'income' | 'expense'
    amountCents: number
    description: string
    frequency: string
    nextDueDate: Date | string
    notes: string | null
  }
}
```

Form fields:
- **Descricao**: `<Input>` text field (required)
- **Conta**: `<select>` from accounts prop (required)
- **Categoria**: `<select>` from categories prop (optional — "Nenhuma" option for null)
- **Tipo**: `<select>` with "Receita" (income) and "Despesa" (expense) — NO transfer option
- **Valor**: `<Input type="number" step="0.01">` — display in BRL (R$), convert to cents on submit: `Math.round(parseFloat(value) * 100)`
- **Frequencia**: `<select>` with options:
  - Diario (daily)
  - Semanal (weekly)
  - Quinzenal (biweekly)
  - Mensal (monthly) — DEFAULT
  - Trimestral (quarterly)
  - Anual (yearly)
- **Data inicio / Proxima data**: `<Input type="date">` — for create mode: label "Data de inicio"; for edit mode: label "Proxima data"
- **Notas**: `<Input>` text field (optional)

Submit behavior:
- Build FormData with all fields; amountCents as integer cents
- nextDueDate formatted as ISO date string
- If `editTemplate` is set: call `updateRecurringTemplate(formData)` with the template id and changed fields
- Otherwise: call `createRecurringTemplate(formData)` with all fields
- On success: `useToast()` success message ("Recorrencia criada" or "Recorrencia atualizada"), call `onClose()`
- On error: `useToast()` error message

Edit mode pre-fill:
- All fields pre-filled from `editTemplate` prop
- Amount displayed as decimal (amountCents / 100)
- Date formatted as YYYY-MM-DD for the date input

Use `useRef<HTMLDialogElement>` for the dialog element. Show with `dialogRef.current?.showModal()` when `open` changes to true. Close on backdrop click (check `e.target === dialogRef.current`).

**C. Create `apps/web/app/(app)/transactions/recurring/page.tsx`** (server component)

```typescript
import { getOrgId, getRecurringTemplates, getUpcomingRecurring, getAccounts, getCategories } from '@/lib/finance/queries'
import { RecurringTemplateList } from '@/components/finance/recurring-template-list'
import { PageHeader } from '@/components/shared/page-header'

export default async function RecurringPage() {
  const orgId = await getOrgId()
  const [templates, upcoming, accounts, categories] = await Promise.all([
    getRecurringTemplates(orgId),
    getUpcomingRecurring(orgId),
    getAccounts(orgId),
    getCategories(orgId),
  ])

  const accountOptions = accounts.map((a) => ({ id: a.id, name: a.name }))
  const categoryOptions = categories.map((c) => ({ id: c.id, name: c.name, type: c.type }))

  return (
    <div className="space-y-6">
      <PageHeader
        title="Transacoes Recorrentes"
        description="Gerencie templates de transacoes recorrentes e gere lancamentos automaticamente"
      />
      <RecurringTemplateList
        templates={templates}
        upcoming={upcoming}
        accounts={accountOptions}
        categories={categoryOptions}
      />
    </div>
  )
}
```
