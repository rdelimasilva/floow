    **supabase/migrations/00001_foundation.sql:**
    Write a comprehensive SQL migration (do NOT use drizzle-kit generate — we need custom SQL for RLS, triggers, and functions that Drizzle cannot express):

    1. **Enums:**
       ```sql
       CREATE TYPE org_type AS ENUM ('personal', 'business');
       CREATE TYPE member_role AS ENUM ('owner', 'admin', 'member', 'viewer');
       CREATE TYPE plan_tier AS ENUM ('free', 'pro');
       CREATE TYPE subscription_status AS ENUM ('active', 'canceled', 'past_due', 'trialing', 'incomplete');
       ```

    2. **Tables:** orgs, profiles, org_members, subscriptions — matching the Drizzle schema exactly. Add `UNIQUE(org_id, user_id)` on org_members.

    3. **Indexes (CRITICAL for RLS performance):**
       ```sql
       CREATE INDEX idx_org_members_user_id ON public.org_members USING btree (user_id);
       CREATE INDEX idx_org_members_org_id ON public.org_members USING btree (org_id);
       CREATE INDEX idx_subscriptions_org_id ON public.subscriptions USING btree (org_id);
       CREATE INDEX idx_profiles_id ON public.profiles USING btree (id);
       ```

    4. **Security definer helper function (performant RLS):**
       ```sql
       CREATE OR REPLACE FUNCTION public.get_user_org_ids()
       RETURNS SETOF uuid
       LANGUAGE sql
       SECURITY DEFINER
       STABLE
       AS $$ SELECT org_id FROM public.org_members WHERE user_id = (select auth.uid()) $$;
       ```

    5. **RLS policies on EVERY table:**
       - Enable RLS on orgs, profiles, org_members, subscriptions.
       - Pattern for orgs: `USING (id IN (SELECT public.get_user_org_ids()))` for SELECT. Owner-only for INSERT/UPDATE/DELETE.
       - Pattern for org_members: `USING (org_id IN (SELECT public.get_user_org_ids()))`.
       - Pattern for profiles: `USING (id = (SELECT auth.uid()))` — profiles are user-scoped, not org-scoped.
       - Pattern for subscriptions: `USING (org_id IN (SELECT public.get_user_org_ids()))`.
       - IMPORTANT: Always use `(select auth.uid())` with select-wrapping for performance, never bare `auth.uid()`.

    6. **on_auth_user_created trigger:**
       Create the function `public.handle_new_user()` as SECURITY DEFINER:
       - Generate org_id UUID
       - INSERT into orgs (personal org with user's name or 'Personal')
       - INSERT into profiles (user_id, email, full_name from raw_user_meta_data)
       - INSERT into org_members (org_id, user_id, role='owner')
       - INSERT into subscriptions (org_id, plan_tier='free', status='active') — every org starts with free plan
       - Wrap in EXCEPTION WHEN OTHERS handler to prevent signup failures (log error but don't block)
       - Create trigger: `AFTER INSERT ON auth.users FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user()`

    7. **Custom Access Token Hook (for JWT org_id claims):**
       Create function `public.custom_access_token_hook(event jsonb)` that injects user's org_ids into JWT app_metadata. Follow the pattern from RESEARCH.md exactly.
       - Grant execute to supabase_auth_admin
       - Revoke from public, anon, authenticated (security best practice)
